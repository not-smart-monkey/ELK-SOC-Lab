# Лабораторная работа: Развертывание и мониторинг безопасности с использованием ELK Stack

## 1. Описание лаборатории (Description)

Целью данной лабораторной работы является развертывание полноценной инфраструктуры для централизованного сбора, анализа и визуализации логов, а также отработка навыков обнаружения вторжений и реагирования на инциденты.

В рамках лаборатории имитируется рабочая среда малого предприятия, где:

- **ELK Stack** (Elasticsearch, Logstash, Kibana) выступает в роли центрального сервера логирования и SIEM-системы.
- **Windows 11** представляет собой рабочую станцию конечного пользователя, на которой генерируются события безопасности.
- **Kali Linux** используется как машина злоумышленника для проведения контролируемых атак.
- **SOC Workstation** - рабочее место аналитика безопасности для мониторинга и расследования инцидентов.

## 2. Архитектура лаборатории (Architecture)

### 2.1. Компоненты лаборатории

| Роль | Хостнейм | ОС | Ключевое ПО |
| :--- | :--- | :--- | :--- |
| **ELK Server** | `elk-lab` | Ubuntu 24.04 LTS | Elasticsearch, Logstash, Kibana |
| **Workstation** | `win11-client` | Windows 11 | Elastic Agent, Sysmon |
| **Attacker** | `kali-attacker` | Kali Linux | Nmap, Hydra, Metasploit, Netcat |
| **SOC Analyst** | `soc-analyst` | Ubuntu 22.04 LTS | Браузер (доступ к Kibana) |

### 2.2. Сетевое взаимодействие

Все компоненты используют единую виртуальную сеть VirtualBox типа **NAT Network** (`NatNetwork-SOC`).

**Подсеть:** `10.0.2.0/24`

| Компонент | Статический IP | Назначение |
| :--- | :--- | :--- |
| **Шлюз (VirtualBox)** | `10.0.2.1` | Выход в интернет через хост |
| `elk-lab` | `10.0.2.10` | ELK Server |
| `win11-client` | `10.0.2.20` | Windows 11 Workstation |
| `soc-analyst` | `10.0.2.30` | SOC Workstation |
| `kali-attacker` | `10.0.2.40` | Kali Linux Attacker |

**Примечание:** В лаборатории используются **статические IP-адреса** для обеспечения стабильной связности между компонентами. Встроенный DHCP-сервер VirtualBox в сети NAT Network **отключён** во избежание конфликтов и непредсказуемого поведения.

### Настройка сети в VirtualBox

1. В `Менеджере сетей` → `NAT Networks` → `NatNetwork-SOC`:
   - **IPv4 Prefix:** `10.0.2.0/24`
   - **Enable DHCP:** **Отключить**
2. На каждой ВМ в настройках сети выбрать `NAT Network` → `NatNetwork-SOC`.
3. Статические IP настраиваются внутри каждой ОС (см. файл `2. Preparation.md`).

### 2.3. Используемые порты

| Источник | Получатель | Порт | Протокол | Описание |
| :--- | :--- | :--- | :--- | :--- |
| `10.0.2.20` (Elastic Agent) | `10.0.2.10` (Logstash) | `5044` | TCP | Отправка логов |
| `10.0.2.10` (Logstash) | `10.0.2.10` (Elasticsearch) | `9200` | HTTPS | Передача обработанных данных |
| `10.0.2.30` (SOC Workstation) | `10.0.2.10` (Kibana) | `5601` | HTTP | Доступ к веб-интерфейсу |
| `10.0.2.40` (Kali) | `10.0.2.20` (Windows 11) | Разные | TCP/UDP | Имитация атак |

## 3. Потоки данных (Data Flow)

1. **Генерация событий:** На **Windows 11** (`10.0.2.20`) служба **Sysmon** и **Elastic Agent** фиксируют события безопасности и системные логи Windows.

2. **Транспортировка в Logstash:** **Elastic Agent** отправляет собранные данные в **Logstash** на `10.0.2.10:5044`.

3. **Обработка в Logstash:** **Logstash** принимает данные и направляет их в **Elasticsearch** на `localhost:9200`.

4. **Индексация и хранение:** **Elasticsearch** индексирует данные и сохраняет их.

5. **Визуализация и анализ:** Аналитик с **SOC Workstation** (`10.0.2.30`) подключается к **Kibana** на `http://10.0.2.10:5601`.

6. **Имитация атак:** **Kali Linux** (`10.0.2.40`) атакует **Windows 11** (`10.0.2.20`). Аномальный трафик фиксируется Sysmon и отображается в Kibana.

## 4. Ресурсы виртуальных машин

Хостовая система: **12 ядер CPU, 16 ГБ RAM, Arch Linux on Hyprland**

| Виртуальная машина | Роль | CPU | RAM | Диск | Обоснование |
| :--- | :--- | :---: | :---: | :---: | :--- |
| `elk-lab` | ELK Server | 4 vCPU | 6 ГБ | 40 ГБ | Elasticsearch (heap 4 ГБ) + Logstash + Kibana. Требователен к памяти и CPU для индексации логов |
| `win11-client` | Windows 11 Workstation | 2 vCPU | 4 ГБ | 40 ГБ | Минимальные требования Win11 + Sysmon + Elastic Agent для генерации событий безопасности |
| `soc-analyst` | SOC Workstation | 2 vCPU | 2 ГБ | 30 ГБ | Ubuntu 22.04 + браузер для Kibana. Достаточно для мониторинга дашбордов |
| `kali-attacker` | Kali Linux | 2 vCPU | 2 ГБ | 40 ГБ | OC + инструменты тестирования (Nmap, Metasploit, Hydra) |
| **Хост** | Arch Linux | 2 vCPU | 2 ГБ | — | Остаток ресурсов для хостовой системы |

> **Примечание по Elasticsearch:** В файле `/etc/elasticsearch/jvm.options` рекомендуется установить `-Xms4g -Xmx4g` для стабильной работы.


## 5. Примечания

- **Расширение:** Можно добавить хосты на Astra Linux, Windows Server, а также развернуть контроллер домена Active Directory (в текущей версии AD отсутствует, Windows 11 использует локальные учётные записи).

- **Кластер Elasticsearch:** Сейчас `single-node`. Можно расширить до 2-3 узлов для опыта работы с кластером.

- **Docker vs Прямая установка:** Лаборатория использует прямую установку компонентов для лучшего понимания архитектуры ELK. Применение Docker Compose позволило бы сэкономить 2-3 ГБ RAM (нет накладных расходов на отдельные ядра ОС), но усложнило бы понимание внутренних процессов.

- **Векторы атак:** Kali атакует только Windows 11. Можно расширить на SOC Workstation или ELK Server.
