Установка и настройка Logstash

В этом разделе описана установка и базовая настройка Logstash на виртуальной машине `elk-lab`.

### Примечания перед началом

- Logstash устанавливается на ту же ВМ, что и Elasticsearch и Kibana (`elk-lab`).
- Logstash будет принимать данные от **Elastic Agent** (порт `5044`) и отправлять их в Elasticsearch (порт `9200`).
- Elastic Agent будет установлен на Windows 11 позже.

[Официальная документация Logstash](https://www.elastic.co/guide/en/logstash/current/installing-logstash.html)

## 1. Установка Logstash

Logstash устанавливается из того же репозитория, что и Elasticsearch и Kibana.

```bash
sudo apt update && sudo apt install -y logstash
```

## 2. Настройка Logstash для приёма данных от Elastic Agent
### 2.1. Создание конфигурационного файла
Создайте единый файл конфигурации для приёма данных от Elastic Agent и отправки в Elasticsearch:
```bash
sudo nano /etc/logstash/conf.d/elastic-agent.conf
```
### 2.2. Содержимое конфигурационного файла
```text
input {
  elastic_agent {
    port => 5044
  }
}

filter {
}

output {
  elasticsearch {
    hosts => ["https://localhost:9200"]
    data_stream => true
    ssl_enabled => true
    ssl_certificate_authorities => ["/etc/elasticsearch/certs/http_ca.crt"]
    ssl_verification_mode => "full"
    user => "elastic"
    password => "ВАШ_ПАРОЛЬ_ELASTIC"
  }
}
```

**Важно**: Замените ВАШ_ПАРОЛЬ_ELASTIC на реальный пароль пользователя elastic, который был сохранён при установке Elasticsearch.
**Важно о безопасности:**
- Пароль хранится в открытом виде только для учебных целей.
- В production-среде используйте Logstash Keystore или переменные окружения.
- Пример безопасного хранения: password => "${ES_PASSWORD}" (после настройки keystore).

## 3. Проверка конфигурации
Чтобы Logstash мог проверить сертификат Elasticsearch, добавьте пользователя logstash в группу elasticsearch:
```bash
# Добавить logstash в группу elasticsearch
sudo usermod -a -G elasticsearch logstash
```
**Примечание**: Без этих прав Logstash не сможет прочитать сертификат, и проверка SSL завершится ошибкой.

Перед запуском проверьте, что конфигурация синтаксически верна:
```bash
sudo -u logstash /usr/share/logstash/bin/logstash --config.test_and_exit -f /etc/logstash/conf.d/ --path.settings /etc/logstash
```
При успешной проверке вы увидите:
```text
Configuration OK
```
## 4. Запуск и добавление в автозагрузку
```bash
sudo systemctl daemon-reload
sudo systemctl enable --now logstash.service
sudo systemctl status logstash.service
```
Убедитесь, что статус — active (running).

**Примечание**: Первый запуск Logstash может занять 1-2 минуты из-за компиляции Java-кода.

## 5. Проверка работы Logstash
### 5.1. Проверка логов Logstash
```bash
sudo journalctl -u logstash --since "2 minutes ago" --no-pager
```

### 5.2. Проверка открытого порта
Убедитесь, что Logstash слушает порт 5044:
```bash
sudo ss -tlnp | grep 5044
```

## 6. Итог
- Logstash установлен из официального репозитория
- Настроен input elastic_agent на порту 5044
-  Настроен output в Elasticsearch с аутентификацией и TLS
- Конфигурация проверена и корректна
- Служба запущена и добавлена в автозагрузку
- Порт 5044 открыт и слушает входящие соединения
