Установка и настройка Elastic Agent на Windows 11

В этом разделе описана установка и настройка **Elastic Agent** на виртуальной машине `win11-client` для сбора и отправки логов в Logstash.

### Примечания перед началом

- Elastic Agent — это универсальный агент для сбора логов, метрик и данных безопасности.
- В учебной лаборатории используется **standalone режим** (без Fleet Server) для простоты и лучшего понимания архитектуры.
- В production-среде рекомендуется использовать **Fleet-managed режим** для централизованного управления агентами.
- **Важно:** Elastic Agent требует прав администратора для сбора системных логов и событий безопасности.
- Данные отправляются в **Logstash** на порт `5044`.

[Официальная документация Elastic Agent](https://www.elastic.co/guide/en/fleet/current/elastic-agent-installation.html)
## 1. Установка Sysmon

Sysmon (System Monitor) — это утилита от Microsoft Sysinternals, которая предоставляет детальную информацию о процессах, сетевых соединениях и изменениях в реестре.

### 1.1. Скачивание Sysmon

1. Откройте браузер на Windows 11.
2. Перейдите на страницу [Sysinternals](https://learn.microsoft.com/en-us/sysinternals/downloads/sysmon).
3. Скачайте архив `Sysmon.zip`.
4. Распакуйте архив в папку `C:\Sysmon`.

### 1.2. Скачивание конфигурационного файла

Для Sysmon требуется конфигурационный файл с правилами логирования. Рекомендуется использовать популярный конфиг **SwiftOnSecurity**:

1. Откройте PowerShell **от имени администратора**.
2. Выполните команды:

```powershell
# Создать папку (если ещё не создана)
New-Item -ItemType Directory -Path "C:\Sysmon" -Force

# Скачать конфигурационный файл
Invoke-WebRequest -Uri "https://raw.githubusercontent.com/SwiftOnSecurity/sysmon-config/master/sysmonconfig-export.xml" -OutFile "C:\Sysmon\sysmonconfig.xml"
```

### 1.3. Установка Sysmon
В том же окне PowerShell (от администратора):
```powershell
cd C:\Sysmon
.\Sysmon64.exe -accepteula -i sysmonconfig.xml
```
После установки появится сообщение об успешной установке.

### 1.4. Проверка работы Sysmon
```powershell
Get-Service Sysmon64
```
Статус должен быть `Running`.

## 2. Установка Elastic Agent
### 2.1. Скачивание Elastic Agent
1. Откройте PowerShell от имени администратора.
2. Создайте директорию и скачайте агент:
```powershell
# Создать директорию
New-Item -ItemType Directory -Path "C:\Elastic-Agent" -Force
cd C:\Elastic-Agent

# Скачать Elastic Agent (версия 9.3.3) (403 for Russia, use VPN)
# Версия 9.3.3, должна совпадать со всеми остальными версиями ELK Stack
Invoke-WebRequest -Uri "https://artifacts.elastic.co/downloads/beats/elastic-agent/elastic-agent-9.3.3-windows-x86_64.zip" -OutFile "elastic-agent.zip"

# Распаковать
Expand-Archive -Path "elastic-agent.zip" -DestinationPath "."
cd elastic-agent-9.3.3-windows-x86_64
```

## 2.2. Установка Elastic Agent
```powershell
.\elastic-agent.exe install
```
Elastic Agent will be installed at `C:\Program Files\Elastic\Agent` and will run as a service. Do you want to continue? [Y/n]: `Y`   
Do you want to enroll this Agent into Fleet? [Y/n]: `n`   
**Примечание**: Команда install автоматически устанавливает Elastic Agent как службу Windows и запускает её. Агент будет автоматически запускаться при перезагрузке системы.

## 3. Настройка Elastic Agent
### 3.1. Получение IP-адреса `elk-lab`
На ВМ elk-lab выполните:
```bash
ip a | grep "inet " | grep -v 127.0.0.1
```
Пример вывода: inet 10.0.2.15/24. Запомните этот IP-адрес — он понадобится для конфигурации.
### 3.2. Создание конфигурационного файла
Создайте или отредактируйте файл `C:\Elastic-Agent\elastic-agent-9.3.3-windows-x86_64\elastic-agent.yml`:
```powershell
notepad C:\Elastic-Agent\elastic-agent-9.3.3-windows-x86_64\elastic-agent.yml
```
Содержимое файла:
```yaml
# ====================================
# Настройки Elastic Agent
# ====================================

# Режим работы: standalone (без Fleet Server)
agent.mode: standalone

# Отправка данных в Logstash
outputs:
  default:
    type: logstash
    hosts: ["10.0.2.15:5044"]
    ssl.enabled: false

# Мониторинг самого агента
agent.monitoring:
  enabled: true
  logs: true
  metrics: true

# ====================================
# Интеграции (сбор данных)
# ====================================

inputs:
  # Системные логи Windows
  - type: winlog
    id: windows-logs
    enabled: true
    name: "Windows-Event-Logs"
    providers:
      - name: Application
      - name: System
      - name: Security
    data_stream:
      dataset: windows.event

  # Логи Sysmon
  - type: winlog
    id: sysmon-logs
    enabled: true
    name: "Sysmon-Event-Logs"
    channel: "Microsoft-Windows-Sysmon/Operational"
    data_stream:
      dataset: sysmon.event
```
**Важно**: Замените `10.0.2.15` на реальный IP-адрес вашей ВМ `elk-lab`.

### 3.3. Перезапуск Elastic Agent
После изменения конфигурации перезапустите службу:
```powershell
Restart-Service elastic-agent
```

## 4. Проверка статуса
```powershell
Get-Service elastic-agent
```
Статус должен быть `Running`.
### 4.1. Просмотр логов агента (если что-то не работает)
```powershell
Get-Content "C:\Elastic-Agent\elastic-agent-9.3.3-windows-x86_64\logs\elastic-agent.log" -Tail 50
```




5. Проверка поступления данных в Kibana
5.1. Открыть Kibana

С машины soc-analyst (или хоста) откройте браузер и перейдите:
text

http://IP_АДРЕС_elk-lab:5601

Войдите с учётными данными elastic и вашим паролем.
5.2. Создать Data View (Index Pattern)

    Перейдите в Stack Management → Data Views.

    Нажмите Create data view.

    Name: windows-logs

    Index pattern: logs-windows.event-*

    Timestamp field: @timestamp

    Нажмите Save data view to Kibana.

Повторите для Sysmon:

    Name: sysmon-logs

    Index pattern: logs-sysmon.event-*

5.3. Просмотр логов в Discover

    Перейдите в Discover.

    Выберите созданный Data View (windows-logs или sysmon-logs).

    Вы должны увидеть поступающие события.
