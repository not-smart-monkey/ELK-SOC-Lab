# Подготовка лабораторной среды (Preparation)

В этом разделе описаны шаги по созданию и базовой настройке всех виртуальных машин перед установкой целевого программного обеспечения.

## 1. Создание виртуальной сети в VirtualBox
**Настройка NAT Network:** 
1. Откройте VirtualBox.
2. Перейдите: `Файл` → `Инструменты` → `Менеджер сетей`.
3. Перейдите на вкладку **NAT Networks**.
4. Нажмите кнопку **Создать**.
5. Параметры:
   - **Имя:** `NatNetwork-SOC`
   - **CIDR сети:** `10.0.2.0/24`
   - **Поддержка DHCP:** **Отключить** (будем использовать статические IP)  
6. Нажмите **Применить** и **OK**.

**Результат:** Создана изолированная сеть, в которой все ВМ будут видеть друг друга и иметь доступ в интернет через хост. DHCP отключён для использования статических IP-адресов.

## 2. Создание виртуальных машин

### Параметры ВМ 

| ВМ | ОС | CPU | RAM | Диск | Статический IP |
|:---|:---|:---:|:---:|:---:|:---:|
| `elk-lab` | Ubuntu 24.04 | 4 | 6 ГБ | 40 ГБ | `10.0.2.10` |
| `win11-client` | Windows 11 | 2 | 4 ГБ | 40 ГБ | `10.0.2.20` |
| `soc-analyst` | Ubuntu 22.04 | 2 | 2 ГБ | 30 ГБ | `10.0.2.30` |
| `kali-attacker` | Kali Linux | 2 | 2 ГБ | 40 ГБ | `10.0.2.40` |

### Общие настройки для всех ВМ

- **Сеть:** Адаптер 1 → `NAT Network` → `NatNetwork-SOC`
- **Видеопамять:** 128 МБ
- **Графический контроллер:** VMSVGA
- **Ускорение 3D:** Включить

### Особенности для Windows 11

- **TPM:** Включить (Система → TPM)
- **Secure Boot:** Включить

## 3. Установка ОС

### Ubuntu / Kali (Linux)

1. Загрузиться с ISO
2. Выбрать **Minimal installation**
3. Имя компьютера и пользователя — любые (например, `user` / `user`)
4. После установки выполнить:

```bash
sudo apt update && sudo apt upgrade -y
```

### Windows 11

1. Загрузиться с ISO
2. При запросе аккаунта Microsoft выбрать «Автономная учётная запись»
3. Имя пользователя: `user`, пароль: `user`
4. После установки отключить визуальные эффекты:  
    - `Параметры` → `Система` → `О системе` → `Дополнительные параметры системы` → `Быстродействие` → `Обеспечить наилучшее быстродействие`


## 4. Настройка статических IP-адресов

**Важно**: Так как DHCP в сети NAT Network отключён, необходимо вручную настроить статические IP-адреса на каждой ВМ.

**ВНИМАНИЕ:** Самый простой способ настроить сеть - через GUI. Ниже настройки если вдруг вы **НЕ** ползуетесь графическим окружением.


### 4.1. Настройка на Ubuntu (`elk-lab` и `soc-analyst`)
**Шаг 1: Узнать имя сетевого интерфейса**
```bash
ip a
```
Обычно это enp0s3 или eth0.

**Шаг 2: Редактирование конфигурации Netplan**
```bash
sudo nano /etc/netplan/01-network-manager-all.yaml
```
Для `elk-lab` (10.0.2.10):
```yaml
network:
  version: 2
  renderer: NetworkManager
  ethernets:
    enp0s3:  # Замените на имя вашего интерфейса
      dhcp4: no
      addresses:
        - 10.0.2.10/24
      routes:
        - to: default
          via: 10.0.2.1
      nameservers:
        addresses: [8.8.8.8, 1.1.1.1]
```
Для `soc-analyst` (10.0.2.30):
```yaml
network:
  version: 2
  renderer: NetworkManager
  ethernets:
    enp0s3:  # Замените на имя вашего интерфейса
      dhcp4: no
      addresses:
        - 10.0.2.30/24
      routes:
        - to: default
          via: 10.0.2.1
      nameservers:
        addresses: [8.8.8.8, 1.1.1.1]
```
**Шаг 3: Применить настройки**
```bash
sudo netplan apply
```
**Шаг 4: Проверить IP**
```bash
ip a
```
### 4.2. Настройка на Kali Linux (10.0.2.40)
**Шаг 1: Редактирование файла interfaces**
```bash
sudo nano /etc/network/interfaces
```

Добавить или изменить:
```text
auto eth0
iface eth0 inet static
    address 10.0.2.40
    netmask 255.255.255.0
    gateway 10.0.2.1
    dns-nameservers 8.8.8.8 1.1.1.1
```
**Шаг 2: Перезапустить сеть**
```bash
sudo systemctl restart networking
# или перезагрузить ВМ
sudo reboot
```

**Шаг 3: Проверить IP**
```bash
ip a
```

### 4.3. Настройка на Windows 11 (10.0.2.20)
1. Нажмите Win + R, введите `ncpa.cpl` и нажмите Enter.
2. Найдите основной сетевой адаптер (обычно "Ethernet").
3. Нажмите правой кнопкой мыши → Свойства.
4. Выберите IP версии 4 (TCP/IPv4) → Свойства.
5. Выберите Использовать следующий IP-адрес и заполните:
    - IP-адрес: 10.0.2.20
    - Маска подсети: 255.255.255.0
    - Основной шлюз: 10.0.2.1
    - Предпочитаемый DNS-сервер: 8.8.8.8
    - Альтернативный DNS-сервер: 1.1.1.1
6. Нажмите ОК и закрыть все окна.

**Проверить IP:**
```cmd
ipconfig
```

## 5. Гостевые дополнения VirtualBox
### На Linux
```bash
sudo apt install -y build-essential dkms linux-headers-$(uname -r)
```

Затем в меню VirtualBox: `Устройства` → П`одключить образ диска Дополнений гостевой ОС` → следовать инструкциям(запустить autorun.sh) → sudo reboot

### На Windows

`Устройства` → `Подключить образ диска Дополнений гостевой ОС` → запустить установщик с виртуального диска → перезагрузить.

## 6. Проверка сети
```bash
# Проверка интернета
ping 8.8.8.8
ping google.com

# Проверка связи с другими ВМ
ping 10.0.2.10   # elk-lab
ping 10.0.2.20   # win11-client(Windows по умолчанию не отвечает на ping)
ping 10.0.2.30   # soc-analyst
ping 10.0.2.40   # kali-attacker
```

### Возможная проблема: DNS не работает

**Симптом:** `ping 8.8.8.8` работает, но `ping ya.ru` или `ping google.com` - нет.

**Решение для Ubuntu / Kali:**

1. Открыть конфигурационный файл `sudo nano /etc/systemd/resolved.conf`
2. Раскомментировать и изменить строки:
```text
DNS=8.8.8.8
FallbackDNS=1.1.1.1
```
3. Перезапустить сервис:
```bash
sudo systemctl restart systemd-resolved
```
**Решение для Windows:**
Обычно проблема не возникает. Если возникает — прописать `8.8.8.8` в свойствах сетевого адаптера вручную. Делалось выше
